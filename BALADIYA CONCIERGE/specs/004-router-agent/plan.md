# Implementation Plan: Router & Agent

**Branch**: `004-router-agent` | **Date**: 2026-06-02 | **Spec**: [spec.md](./spec.md)

## Summary

Build the classifier-driven router that handles easy turns via workflow (no agent LLM call) and routes ambiguous turns to a bounded tool-calling agent with three tenant-scoped tools (`rag_search`, `capture_request`, `escalate`) and Redis-backed session memory.

## Technical Context

**Language/Version**: Python 3.11

**Primary Dependencies**: FastAPI, httpx (LLM hosted API + modelserver), redis (async), SQLAlchemy (async), pydantic, tenacity

**Storage**: PostgreSQL (`capture_requests`, `escalation_tickets`); Redis (session memory + rate limits)

**Testing**: pytest + pytest-asyncio; LLM and modelserver mocked for unit tests; 15-example tool-selection eval set

**Target Platform**: `api` Docker service

**Performance Goals**: Agent loop < 5s p95 for single-tool turns; workflow path < 200ms p95

**Constraints**: `max_tool_calls` configurable (default 3, set via `Settings.max_tool_calls`); `max_tokens_per_turn` configurable; `capture_request` always schema-validated before write; session key = `session:{session_id}:{tenant_id}` (NEVER just `session:{session_id}`)

## Constitution Check

- [x] `capture_request` tenant_id from session context (JWT), never from tool payload
- [x] `rag_search` always tenant-filtered
- [x] Agent loop bounded (max_tool_calls + max_tokens_per_turn)
- [x] Spam dropped before any write — classifier gates `capture_request`
- [x] Session key scoped to both session_id AND tenant_id

## Router Decision Logic

```
POST /chat (resident message)
  ↓
guardrails.validate(message)          # stub (passthrough) in phase 004; full sidecar wired in phase 005
  ↓
lang, variety = lang_detect(message)  # phase 007 — defaults to en if absent
  ↓
result = modelserver.classify(message)
  ↓
if result.confidence < threshold[result.intent]:
    → agent_service.run(message, session)   # hard path

if result.intent == "spam":   → drop (return empty 200)
if result.intent == "human":  → escalate tool directly
if result.intent == "report": → capture_request tool directly
if result.intent == "question": → rag_search → compose answer
```

## Agent Bounded Loop

```python
async def run(message, session, tenant_id, max_tool_calls=3):
    turns = session.get_turns()
    for i in range(max_iterations):
        response = await llm.complete(system_prompt, turns + [message], tools=TOOLS)
        if response.stop_reason == "end_turn":
            return response.text
        tool_result = await dispatch_tool(response.tool_call, tenant_id)
        turns.append(tool_result)
    # cap exceeded → auto-escalate
    await escalate_tool.run(reason="max_iterations_exceeded", tenant_id=tenant_id)
    return FALLBACK_MESSAGE
```

## Session Memory Key Pattern

```
session:{session_id}:{tenant_id}   TTL: 1800s (30 min)
```

The session_id is a UUID generated by the widget at conversation start and passed in each request. The tenant_id is always from the JWT token.
